# Contributing

This is going to be quick since I don't imagine many people will be working with this.

Guidelines:
1. Be polite and courteous.
1. Try to follow the style of code and commits as used in the repo.
1. Write all code, documentation, commit messages, etc. by yourself, or cite sources if you do not. This means no code from LLMs or similar softwares.

One thing I would recommend is running `git update-index --assume-unchanged settings/selected_profile.txt` so that changes to selected_profile.txt are not tracked.
